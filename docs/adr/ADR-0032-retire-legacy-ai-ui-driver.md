# ADR-0032: Вывод legacy-драйвера AI UI

## Статус

Accepted

## Дата

2026-07-04

## Контекст

После ADR-0027 и ADR-0028 в проекте одновременно поддерживаются три драйвера ИИ-чата,
выбираемые настройкой `LOCAL_BUSINESS_AI_UI_DRIVER`:

- `native` — основной самописный AG-UI-совместимый чат (режим по умолчанию);
- `copilotkit` — пилотный драйвер и эталон совместимости AG-UI;
- `legacy` — старый интерфейс чата, предшествовавший protocol foundation.

Каждый драйвер — это отдельная ветка шаблонов, статики, режимов запуска и строка в матрице
тестов. Архитектурное ревью 2026-07-04 отметило тройную матрицу как источник лишней
стоимости сопровождения. `native` признан целевым и является режимом по умолчанию;
`legacy` не имеет собственной ценности — он сохранялся только как страховка на время
становления protocol foundation.

Владелец проекта 2026-07-04 принял решение вывести `legacy`; по CopilotKit-драйверу
решение осознанно отложено.

## Решение

1. Драйвер `legacy` выводится из кода:
   - значение `legacy` удаляется из допустимых для `LOCAL_BUSINESS_AI_UI_DRIVER`
     (`config/settings.py`); при обнаружении `legacy` в env конфигурация падает с понятным
     сообщением о переходе на `native`;
   - удаляются шаблоны, статика, view-ветки и тесты, используемые только `legacy`-драйвером;
   - из документации (`docs/guides/AI_UI_PROTOCOL_OPERATIONS.md`, deployment-инструкции,
     README) убираются упоминания `legacy` как рабочего режима.
2. Драйвер `copilotkit` сохраняется без изменений статуса: равноправный драйвер, пилот и
   эталон совместимости AG-UI (ADR-0027). Решение о его дальнейшей судьбе будет принято
   отдельно и оформлено отдельным ADR; в рамках текущего решения его контур не трогать.
3. Матрица проверок AI UI сокращается с `legacy/copilotkit/native` до `copilotkit/native`.

## Альтернативы

- **Оставить `legacy` как аварийный fallback.** Отклонено: `native` уже режим по умолчанию
  с собственным e2e-покрытием; fallback-ветка без регулярных прогонов деградирует и создаёт
  ложное чувство страховки.
- **Вывести одновременно `legacy` и `copilotkit`.** Отклонено владельцем: CopilotKit
  остаётся эталоном совместимости AG-UI и источником проверенных UX/runtime-паттернов,
  решение по нему не принято.

## Последствия

Положительные:

- матрица драйверов и тестов сокращается на треть;
- меньше шаблонов и статики в сопровождении, меньше веток в `apps/ai/views.py`
  и `apps/ai/ui_runtime/`;
- упрощаются deployment-инструкции.

Отрицательные и риски:

- установка, у которой в `.env` задан `LOCAL_BUSINESS_AI_UI_DRIVER=legacy`, не поднимется
  после обновления — сообщение об ошибке обязано явно указывать замену (`native`);
- откат возможен только через git-историю.

## Реализационные правила

- Работы идут в пакете 09 блока `workflow/active/architecture-review-remediation-2026-07-04/`.
- Перед удалением — инвентаризация ссылок на `legacy` по коду, шаблонам, статике, тестам,
  документации и deployment-профилям (включая `LOCAL_BUSINESS_AI_UI_DRIVER_EXPLICIT`-логику).
- Уточняет ADR-0028: допустимая матрица драйверов становится `copilotkit | native`.
  ADR-0027 (CopilotKit) не меняется.
