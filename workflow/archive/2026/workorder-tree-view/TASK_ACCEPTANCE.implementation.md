# Task Acceptance: implementation

Дата: 2026-05-30.

## Принято

MVP реализации дерева заявок принят на уровне локальной проверки.

Проверено:

- режимы `view=board` и `view=tree` работают на существующей странице заявок;
- дерево строится поверх серверно видимых заявок;
- заказчик ограничен веткой своего `User.department`;
- формы создания и редактирования не дают выбрать чужие подразделения и медизделия;
- правый сайдбар используется для деталей, создания и редактирования;
- стили дерева сохраняют визуальный язык доски;
- добавлены unit/view и Playwright e2e проверки.

## Условия финальной приемки владельцем

- Выполнить браузерный e2e на рабочем стенде с учетными данными `E2E_USERNAME` и `E2E_PASSWORD`.
- Проверить реальные учетные записи заказчиков на заполненность `User.department`.
- Проверить, что исторические заявки не требуют отдельного миграционного правила видимости.
