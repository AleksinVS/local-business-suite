# Уведомления пользователя

## Назначение

Портал показывает уведомления о событиях заявок: новая заявка, изменение, назначение, комментарий, смена статуса и закрытие.

Первый этап работает без стороннего Web Push. Это значит, что системные уведомления браузера приходят только пока портал открыт во вкладке или в установленном PWA-окне. Если портал закрыт, события сохраняются на сервере и будут видны в центре уведомлений при следующем входе.

## Центр уведомлений

Центр уведомлений находится в верхней панели портала.

В нем доступны:

- счетчик непрочитанных уведомлений;
- список последних событий;
- переход к объекту портала;
- действия "Прочитано" и "Скрыть";
- включение или выключение браузерных уведомлений для текущего устройства.

Пользователь видит только те уведомления, которые сервер разрешил его учетной записи. Ссылка из уведомления не заменяет проверку прав: при открытии заявки портал снова проверяет доступ.

## Включение браузерных уведомлений

1. Войдите в портал.
2. Откройте "Уведомления" в верхней панели.
3. Нажмите "Включить".
4. Подтвердите разрешение в системном окне браузера.

Портал не запрашивает разрешение автоматически при загрузке страницы. Запрос появляется только после нажатия кнопки.

Если браузер или система запретили уведомления, центр уведомлений в портале продолжит работать. Для повторного включения нужно изменить разрешение в настройках браузера или операционной системы.

## Установка как PWA

Если браузер поддерживает установку PWA, портал можно установить как приложение через меню браузера.

Установленное PWA-окно не является отдельным фоновым агентом. Уведомления приходят, пока это окно открыто.

## Какие данные попадают в системное уведомление

Системное уведомление содержит только короткий безопасный текст, например:

```text
Заявка №123 назначена вам
Открыть в портале
```

В текст уведомлений не включаются описания заявок, комментарии, вложения, медицинские данные, сетевые пути, секреты или диагностические подробности.

## Ограничения первого этапа

- Нет системных уведомлений при полностью закрытом браузере.
- Нет Push API, VAPID и сторонних browser push endpoints.
- Поведение зависит от браузера, HTTPS, настроек операционной системы и корпоративных политик.
- На production-стенде для service worker и browser notifications нужен HTTPS. `localhost` допускается только для разработки.

Постоянный фоновый режим планируется отдельным вторым этапом через опциональный Tauri-клиент.
