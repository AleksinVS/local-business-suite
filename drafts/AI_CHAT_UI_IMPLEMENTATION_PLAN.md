# План реализации современного интерфейса ИИ-чата (HTMX + Tailwind + Flowbite)

## Цель
Создать современный, отзывчивый и интуитивно понятный интерфейс ИИ-чата, аналогичный популярным решениям (ChatGPT, Claude), с поддержкой отправки текстовых сообщений и прикрепления файлов (изображений, аудио, документов) без использования тяжелых JS-фреймворков.

## Архитектурный стек
*   **Бэкенд:** Django 5.x
*   **Фронтенд:** HTML, Tailwind CSS, Flowbite (компоненты), HTMX (для динамического обновления и загрузки файлов), SSE (для стриминга ответов ИИ).

---

## Этап 1: Обновление модели данных (Модели и БД)

**Задачи:**
1.  Создать модель `ChatAttachment` в `apps/ai/models.py` для хранения прикрепленных файлов.
    *   Поля: `message` (ForeignKey на `ChatMessage`), `file` (FileField), `file_type` (строка: image, document, audio), `file_name` (строка), `file_size` (число), `created_at`.
    *   *Примечание:* Изначально вложения можно привязывать к сессии (в статусе pending), а после создания сообщения — к конкретному сообщению пользователя.
2.  Создать и применить миграции базы данных.

## Этап 2: Интеграция UI библиотек (Tailwind & Flowbite)

**Задачи:**
1.  Убедиться, что Tailwind CSS настроен для сборки классов, используемых в шаблонах чата. (Если используется `django-tailwind`, обновить конфигурацию).
2.  Подключить стили Flowbite (если необходимо, добавить плагин Flowbite в `tailwind.config.js` или подключить CDN для базовых стилей чата, если не используется сборщик).
3.  Определить базовую структуру страницы чата: сайдбар с историей слева, основное окно чата по центру, панель ввода внизу.

## Этап 3: Разработка шаблонов (HTML + Tailwind/Flowbite)

**Задачи:**
1.  **Бабблы сообщений (Chat Bubbles):** 
    *   Переписать отображение сообщений в `templates/ai/chat_detail.html` с использованием классов Tailwind/Flowbite (`flex`, `justify-end`, `justify-start`, `bg-blue-500`, `rounded-br-none` и т.д.).
    *   Оформить аватарки или иконки для Пользователя и Ассистента.
2.  **Отображение вложений:**
    *   Для **изображений**: сетка миниатюр.
    *   Для **документов**: карточка с иконкой файла, названием и размером.
    *   Для **аудио**: стандартный `<audio controls>`.
3.  **Поле ввода (Chat Input):**
    *   Создать область ввода с кнопками прикрепления файла (скрепка) и отправки.
    *   Добавить зону предварительного просмотра прикрепленных файлов (до отправки).

## Этап 4: Логика работы с файлами (HTMX + JS)

**Задачи:**
1.  **Асинхронная загрузка (HTMX):**
    *   Обновить форму отправки в `templates/ai/chat_detail.html`: добавить `enctype="multipart/form-data"` (или `hx-encoding`).
    *   Настроить HTMX для отправки запроса с файлами.
2.  **Обработка на сервере (Django Views):**
    *   Обновить `AIChatMessageCreateView` в `apps/ai/views.py` для приема файлов из `request.FILES`.
    *   Сохранять файлы, создавать объекты `ChatAttachment` и привязывать их к новому сообщению пользователя.
3.  **Улучшение UX (Vanilla JS):**
    *   Добавить простой скрипт для отображения превью выбранного файла (имя файла или миниатюра картинки) перед отправкой.
    *   Поддержка Drag-and-Drop: перехват событий перетаскивания файла в область чата для автоматического добавления в `input type="file"`.

## Этап 5: Интеграция с Agent Runtime

**Задачи:**
1.  При отправке запроса в `AgentRuntimeClient.chat_stream`, помимо текста `prompt` и `history`, передавать метаданные о прикрепленных файлах (имена, типы, URL/пути).
2.  Адаптировать системный промпт ИИ или логику `build_tools`, чтобы агент знал о наличии вложений и мог их учитывать при ответе (например, если появится навык чтения документов).

## Этап 6: Тестирование и Полировка

**Задачи:**
1.  Проверить стриминг (SSE) — убедиться, что новый дизайн не ломает текущую логику появления текста.
2.  Протестировать загрузку различных типов файлов (картинки, pdf) и их отображение в истории.
3.  Проверить адаптивность (мобильная версия чата).
