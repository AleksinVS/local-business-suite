# Task Acceptance: documentation

Дата: 2026-05-30.

## Принято

Документационный срез принят как готовая основа для реализации.

Проверено:

- есть ADR с контекстом, решением, альтернативами, последствиями и критериями приемки;
- активный план описывает UI/UX, доступ заказчика, архитектуру реализации, этапы и проверки;
- workflow-блок содержит brief, architect plan, task packets, отчет и приемку документационного среза;
- backlog содержит ссылку на задачу как готовую к реализации;
- документация явно фиксирует, что дерево строится поверх серверно видимых заявок.

## До начала реализации

- Согласовать, что виртуальный корень "Организация" достаточен для MVP.
- Проверить реальные данные заказчиков: заполненность `User.department`.
- Подтвердить, что customer branch visibility должна быть строгой и не сохраняет исключение "все authored заявки вне ветки".
