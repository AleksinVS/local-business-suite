# Block brief: memory MVP simplification

## Цель

Упростить MVP системы памяти: сделать `MemoryKnowledgeItem` главным объектом сохраненного знания, убрать `MemoryBelief` из обычного пути и оставить понятные правила надежности источников.

## Объем работ

- закрепить `MemoryKnowledgeItem` как главный объект памяти MVP;
- убрать `MemoryBelief` из обычного сохранения и поиска;
- сохранить текущие каналы поиска: векторный, полнотекстовый и графовый;
- не расширять в этом блоке стратегию графового поиска;
- разделить обработчик очереди памяти и будущую ночную рефлексию;
- упростить статусы надежности источников;
- обновить тесты и документацию.

## Не входит

- перенос истории чата в отдельный SQLite-файл;
- изменение физического хранения сообщений чата;
- детальный план использования графового поиска;
- production UI для проверки знаний;
- полноценная проверка противоречий между источниками;
- новая система хранения секретов.

## Ограничения

- Django остается главным местом политик, прав доступа, служебных данных и журналов.
- Runtime-данные остаются в `data/`.
- Контракты по умолчанию остаются в `contracts/`.
- Нельзя ослаблять проверку прав доступа, чувствительности и надежности источника.
- Нельзя удалять данные пользователя без безопасной миграции.
- Текущие сервисы должны продолжить запускаться.

## Критерии готовности

- `memory.remember` в обычном пути создает/обновляет `MemoryKnowledgeItem`.
- `MemoryBelief` не создается в обычном пути.
- `memory.search` не возвращает `MemoryBelief`.
- Векторный, полнотекстовый и текущий графовый поиск не сломаны.
- Правила надежности источников сведены к понятной MVP-схеме или совместимому отображению.
- Текущая обработка очереди не называется полноценной ночной рефлексией.
- Unit и e2e проверки выполнены.
