# Block brief: исправление разрывов MVP-памяти

## Цель

Исправить три разрыва после упрощения MVP-памяти:

- `memory.remember` должен только ставить задачу в очередь;
- обработанное знание должно находиться через `memory.search`;
- `MemoryBelief` не должен входить в MVP-схему и обычный путь.

## Архитектурные источники

- `docs/adr/ADR-0010-memory-mvp-simplification.md`
- `docs/architecture/MEMORY_MVP_SIMPLIFICATION_PLAN.md`
- `docs/architecture/MEMORY_MVP_REMEDIATION_PLAN.md`
- `docs/planning/active/memory-mvp-remediation.md`

## Объем работ

- вернуть AI-инструмент `memory.remember` к очереди;
- сохранить обработку очереди через совместимую команду;
- индексировать обработанный `MemoryKnowledgeItem` в полнотекстовый поиск;
- добавить тест "поставил в очередь -> обработал -> нашел поиском";
- убрать `MemoryBelief` из MVP-модели/миграции/admin/services/tests или явно вывести за пределы MVP;
- обновить контракты и документацию.

## Не входит

- хранение сообщений чата;
- стратегия графового поиска;
- внешние коннекторы;
- новая система секретов.

## Ограничения

- Не ослаблять права доступа, чувствительность и проверку надежных источников.
- Не индексировать значения секретов.
- Не удалять данные без безопасной миграции.
- Сохранять совместимость команды `memory_reflect_chats` для расписаний.
- Использовать `./.venv/bin/python`, потому что системная команда `python` может отсутствовать.

## Критерии готовности

- `memory.remember` возвращает `queued`.
- Очередь обрабатывается отдельной командой или сервисной функцией.
- После обработки `memory.search` возвращает сохраненный текст.
- `MemoryBelief` не участвует в MVP.
- Контракты инструментов отражают очередь.
- Unit и e2e проверки проходят.
