# Planning System

Этот каталог содержит человекочитаемый контур планирования проекта.

## Назначение

- `backlog.md` — актуальная очередь активных и будущих задач.
- `active/` — планы крупных направлений, которые сейчас прорабатываются или реализуются.
- `archive/` — закрытые или остановленные планы.

Backlog отвечает на вопрос "что делать дальше". Планы отвечают на вопрос "как устроена крупная работа". Workflow-пакеты в `workflow/` отвечают на вопрос "как эта работа была исполнена и принята".

## Правила

1. Не добавляйте архивный раздел в `backlog.md`.
2. Не храните executor reports, acceptance reports и task packets в `docs/planning/`.
3. Для мелких задач достаточно записи в `backlog.md`.
4. Для крупных направлений создавайте отдельный файл в `active/`.
5. После завершения переносите итоговый план в `archive/<YYYY>/` и удаляйте задачу из backlog.
6. Если план фиксирует архитектурное решение, сначала создайте или обновите ADR в `docs/adr/`.

## Когда нужен workflow

Создавайте workflow-блок в `workflow/active/<block-id>/`, если работа multi-step, multi-agent, high-risk или требует task packets и приемочных отчетов.
