# Active Workflow Blocks

Здесь создаются новые workflow-блоки для сложных работ.

Используйте `workflow/active/<block-id>/`, если работа multi-step, multi-agent, high-risk или требует task packets, executor reports и acceptance reports.

Не создавайте новые блоки прямо в корне `workflow/`. Существующие корневые блоки считаются legacy-историей.
