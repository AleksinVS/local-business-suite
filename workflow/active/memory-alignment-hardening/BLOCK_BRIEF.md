# Block Brief: доводка памяти после ревью целевой архитектуры

## Зачем

Блок `memory-hybrid-knowledge-v05-alignment` (ADR-0030) исполнен и принят 2026-07-04. Ревью целевой архитектуры 2026-07-06 нашло пробелы, не покрытые исполнением, а ретроспектива блока зафиксировала остаточные риски. Без доводки: git-autocrlf на Windows ломает content-hash дисциплину; возможна петля `needs-reconcile`; конкурентные запуски reconciler не исключены; словарь рёбер существует без производителя рёбер; критерии возврата отложенных контуров (профили, граф) нефальсифицируемы без golden set.

## Что сделать

Три пакета (`task-packets/`): (01) LF-нормализация канона + канон-ориентированный обход (новые/перенесенные/пропавшие файлы, базисы из git при `--force`, `kind` в дедуп-ключе) + защита reconciler от петли и конкурентных запусков; (02) golden set `memory_eval` + матрица восстановления; (03) инструкция эмиссии `relations:` для семантического прохода с протоколом противоречия + обновление входящих рёбер при `supersede` + команда `memory_impact` (сверка на записи) + чистка dead-config профилей + фикс фильтра review-UI.

Дополнение по итогам кросс-ревью параллельного kb-проекта (2026-07-06): проверка его находок по коду выявила, что обход reconciler идет от проекции, а не от канона (файлы в обход `memory.remember` невидимы, перенос файла молча пропускается), а базисы гарда и очереди живут в сносимой проекции — шаги 10-12 пакета 01 и шаг 7 пакета 03 закрывают это.

Дополнение по итогам обсуждения паттерна «сверка на записи» в том же kb-проекте (2026-07-07, зафиксирован в концепте §7.4): в этой системе роль пишущего-синтезатора несет семантический проход, поэтому протокол противоречия (`contradicts` с провенансом вместо молчаливой правки чужой страницы) ложится в его инструкцию, а детерминированный проверочный набор — в read-only команду `memory_impact` для прямых правок канона (шаги 2 и 8 пакета 03); горячий путь `memory.remember` сверкой не нагружается.

## Границы

- Архитектура ADR-0030 не меняется; это доводка реализации.
- Data store (5а/5б), graph runtime search, профили ранжирования, фрагментный поиск — не в этом блоке.
- Массовая перезапись существующих файлов знаний запрещена.
- Каждый пакет проверяется на SQLite и PostgreSQL.
